How to create a blog for free in minutes (beginners)

April 25, 2024 · James Hicks

I recently spent a day or so researching the best way publish a personal blog for free.

I wanted something fast and easy to use. But I also wanted the flexibility to customise the blog later down the line. Instant publishing platforms like Wordpress seemed too inflexible, so I looked towards other static site generators.

I settled on using Hugo and am happy with the result. Within just a few steps, I published my blog for free on GitHub Pages.

To save you time piecing together the different sources of documentation. Here is the process laid out in full, with additional commentary on things I wish I knew when setting it up myself.

As a bonus at the end, I’ll show you how to point a custom domain to the blog.

Disclaimer: you need to buy a domain for this part 💸. It’s not essential, but it makes your blog look more professional.

Create a personal blog in minutes#

Pre-requisites#

Getting started requires that you have basic familiarity with the command line and have git (version control) installed. This might sounds scary, but I promise it’s easier that you think!

You will also need access to a code editor (I recommend VS Code). So if you need to need to get up to speed, here are some useful links.

  • Getting started with VS Code

  • Command line for beginners

  • Instructions for installing git

Install Hugo#

Now, let’s get started. First, follow the Hugo installation instructions for your platform. For macOS, I use Homebrew from the command line.

brew install hugo

Pick your theme#

Next the fun part! Look through these Hugo themes and pick one that you like for your blog. I went with the PaperMod theme.

There will be installation instructions specific to your chosen theme on its dedicated page.

However, in general, for the next step you just need a link to your theme’s GitHub repositiory. It should look something like this:

https://github.com/adityatelange/hugo-PaperMod

Important: Where I use the link above and references to ‘PaperMod’ in the code below, modify to your chosen theme as appropriate.

Create a blog#

You’re ready to create your blog. Using the command line, move into a directory where you would like to store your blog.

Then run the two commands below on the command line. These will create a new directory titled YOUR-BLOG-DIRECTORY-NAME and move you into that directory.

hugo new site YOUR-BLOG-DIRECTORY-NAME
cd YOUR-BLOG-DIRECTORY-NAME

Next up, these commands initialise git version control and download your chosen theme into your blog directory (remember to use your theme’s URL).

git init
git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

Now add your chosen theme to your hugo.toml (your blog’s main configuration file.)

echo "theme = 'PaperMod'" >> hugo.toml

If everything is set up correctly, hugo server should launch a local version of your blog.

hugo server

You should see something like this in the terminal:

Web Server is available at http://localhost:62168/ (bind address 127.0.0.1)

Use the localhost address in the browser to view your new blog.

Customise the landing page#

Your template is up and running. Now let’s customise the landing page.

Open your blog directory up in your favourite code editor. If you are using VS Code, the command is:

code PATH/TO/YOUR-BLOG-DIRECTORY-NAME

The folder directory should look something like this:

.
├── archetypes
├── assets
├── content
├── data
├── hugo.toml
├── i18n
├── layouts
├── public
├── static
└── themes

Open up the hugo.toml file and add this to the bottom:

[params.homeInfoParams]
Title = "Welcome"
Content = """
👋 Welcome to MY PERSONAL blog!
"""

Now save the file and refresh your browser. You should see the changes on the page.

Add your first post#

The landing page is set. Now to add our first blog post.

hugo new content posts/YOUR-FIRST-POST.md

This will create a markdown file in the content/posts/ directory of your project that looks like this:

+++
title = 'YOUR FIRST POST'
date = 2024-04-26T18:43:42+01:00
draft = false
+++

Go ahead and add some content to your post below the front matter. The post is in draft mode, so to preview, you’ll need to use the --buildDrafts or -D flag when running.

hugo server -D

Further customisation#

Detailed instructions on further customisation is beyond the scope of this article. Just know that Hugo is very flexible!

If you want to make further changes, do not make them directly in the theme directory. Instead, familiarise yourself with the template lookup order and define teamplates that override bits of the theme you would like to change.

Publish to GitHub Pages#

The blog works nicely on your local machine. But how do you publish it on the internet so other people can see it too?

GitHub Pages is certainly one of the easiest ways. And it’s the method we’ll use below.

Set up your GitHub repo#

To deploy a website you need to move your local code into a GitHub repository. There are multiple ways to do this. Use the link to determine the option that suits you best. I like to use the GitHub command line interface.

brew install gh # if you don't have the GitHub CLI installed.
gh repo create hen follow the in line instructions.

Then follow the in line instructions.

Configure GitHub actions#

Once your GitHub repo is created, you need to configure GitHub Actions to automatically deploy your website each time you push updated code to the repo.

Follow the instructions provided in the Hugo documentation for deploying to GitHub pages. These instructions will guide you through:

  1. Setting the GitHub Pages build and deployment source to GitHub actions.
  2. Adding a hugo.yaml file to a .github/workflows folder in your project directory.

Push your site to GitHub#

Finally, add your changes to git and push a commit to your github repo.

git add .
git commit -m 'Deploy blog with GitHub Actions'
git push --set-upstream origin main

That’s it! Wait a minute or so for the GitHub Actions to run. Then navigate back to the Github Pages settings find a URL for your newly deployed and publically available blog 🎉.

View the site in your browser.

BONUS Add your own domain#

After you have settled down from the pure joy and excitement of finally publishing something FOR REAL on the internet (👊🏼), your next thought will likely be about your domain name.

Check the browser address bar and your blog’s URl will look something like:

https://USERNAME.github.io/YOUR-BLOG-DIRECTORY-NAME/

Hmm. Not very professional…

Here’s how to add your own domain name.

Buy your domain#

Go to your favourite domain name registry and purchase a domain. There are plenty to choose from. I use GoDaddy, so will show screenshots of their interface below.

Configure DNS settings#

Once you have purchased your domain, log in to your GoDaddy account and navigate to the DNS records section of the domain.

Here you should configure your DNS records to point to your newly created blog hosted by GitHub Pages.

My account looks like this:

Important: these changes can take up to 48 hours to propagate through the internet. However, often it happens much quicker.

Add Custom Domain to GitHub#

Whilst you wait, add your new domain to the top or your hugo.toml file of your project directory.

baseURL = 'https://YOUR-NEW-DOMAIN.com'

Push your changes to GitHub.

git add .
git commit -m 'Add domain name to config'

Finally, within your GitHub repo, navigate back to the GitHub Pages section of the ‘Settings’ tab and enter your new domain in the space provided.

This will verify that you have set up your DNS records correctly.

Once the DNS changes have fully propogated (remember, be patient!) you will be able to access your blog at your own custom domain. Wahoo! 🎉

You did it! Now, go forth and start blogging.
